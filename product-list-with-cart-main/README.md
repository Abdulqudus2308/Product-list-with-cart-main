## Overview

### The challenge

Users should be able to:

- Add items to the cart and remove them
- Increase/decrease the number of items in the cart
- See an order confirmation modal when they click "Confirm Order"
- Reset their selections when they click "Start New Order"
- View the optimal layout for the interface depending on their device's screen size
- See hover and focus states for all interactive elements on the page

### What I learned

JavaScript / DOM

querySelectorAll and indexing elements correctly across multiple NodeLists
.closest() to traverse up the DOM and find parent elements
stopPropagation() to prevent event bubbling issues
Dynamic rendering — updating the DOM based on cart state without a framework

Data & Fetch API

Fetching from a local data.json file using fetch()
Parsing JSON and mapping over the data to dynamically generate HTML
Connecting fetched data to UI elements reliably using indices

Cart Logic

Building add-to-cart, increment, decrement, and remove functionality from scratch
Keeping UI in sync with an underlying data/state object
Handling edge cases like quantities hitting zero

Event Handling

Delegated vs direct event listeners
Debugging events firing on the wrong elements
Button state management (toggling between "Add to Cart" and the quantity controller)

General Problem Solving

Reading error messages and tracing bugs methodically
Refactoring code when the initial approach breaks down
